Programming the 3x3 Rubik's Cube

For those who are not familiar with the Rubix cube it is a popular puzzle that has many different variations. In this article I want to cover the 3X3 version of the puzzle. There are 6 sides with 3 columns and 3 rows of different colors: yellow, orange, blue, red, green, and white. There are many challenges to programming this from scratch and as mentioned earlier there are many resources online that probably do it better than I can. That being said let’s get started.

First Journey

My first task was to define the cube so I could start playing around with moving it and even try solving it on the screen. I wanted to stay away from anything 3D starting out with, but is definitely something I want to get to long term. I defined the cube the best I could and created algorithms for each move. I did it by describing the cube by each face of the cube, if you are not familiar with cube notation it goes like this. f-front, t-top, l-left, r-right, d-down, b-back.

Solving

Start

  • Check to see if there is at least one piece in the correct layer, but in the wrong place.
    • Turn the layer until the piece is in the correct spot
    • Once we get here we will need algorithms to continue
  • Flip Edge
    F2 D R F' R'
  • If all the pieces are a part of the layer then we can do algorithms to fix the issue.

    • Once the pieces are flipped right side up and part of the same layer we can use the algorithm.

      R L' F2 R' L U2 R L' F2 R' L U2 F2

    • To do a variation of the algorithm above to make it a little more generialized. Set up you're cube so switch to edges at once.

      R L' F2 R' L U R L' F2 R' L U' F2 X2 Repeat

      Long Hand

      R L' F2 R' L U R L' F2 R' L U' F2 L R' B2 R L U L R' B2 L' R U' B2

      If we have to do only three edges. Let's assume clockwise rotation of edges. Make sure solved edge is on the left

      L' R F2 L R' U2 F2 U' R L' F2 R' L U' F2

      ~ repeat two more times if needed

  • Corners

    • Find the correct corner and identify the destination. Rotate the bottom until that corner is where it needs to be. Then preform the following until the corner pops up.

      R' D' R D

      Turn the cube and repeat, or you can use U as well and then use D to locate the next corner

  • Edges of second layer
      Turn Cube and or the U layer to figure out where to place the edge.

    • If you have to move edge to the left use

      U' L' U L U F U' F'

      If you have to move edge to the right use

      U R U' R' U' F' U F

      If all is done correctly the second layer should be complete

  • Solve top layer for cross

    Count the number of edges correctly oriented.

    • If the three edges is in the shape of an L use

      F R U R' U' R U R' U' F'

    • If the three edges are in a stright line use

      F R U R' U' F'
  • Rotate Corners

Code Review

Each turn method looks like this. This is the method for the back-layer turn


    var topTemp = new char[3] { TopLayer[0, 0], 
      TopLayer[0, 1], TopLayer[0, 2] };

    var bottomTemp = new char[3] { DownLayer[2, 0], 
      DownLayer[2, 1], DownLayer[2, 2] };
      
    var leftTemp = new char[3] { LeftLayer[0, 0], 
      LeftLayer[1, 0], LeftLayer[2, 0] };

    var rightTemp = new char[3] { RightLayer[0, 2], 
      RightLayer[1, 2], RightLayer[2, 2] };

    // ---------------------------------------------------------
    var centers = new char[] { BackLayer[1, 0], 
      BackLayer[2, 1], BackLayer[1, 2], BackLayer[0, 1] };

    var corners = new char[] { BackLayer[2, 0], 
      BackLayer[2, 2], BackLayer[0, 2], BackLayer[0, 0] };

    // Corners           
    BackLayer[0, 0] = corners[0];
    BackLayer[2, 0] = corners[1];
    BackLayer[2, 2] = corners[2];
    BackLayer[0, 2] = corners[3];

    // Centers
    BackLayer[0, 1] = centers[0];
    BackLayer[1, 0] = centers[1];
    BackLayer[2, 1] = centers[2];
    BackLayer[1, 2] = centers[3];
    // ---------------------------------------------------------

    // right to top
    TopLayer[0, 0] = rightTemp[0];
    TopLayer[0, 1] = rightTemp[1];
    TopLayer[0, 2] = rightTemp[2];

    // bottom to right            
    RightLayer[0, 2] = bottomTemp[2];
    RightLayer[1, 2] = bottomTemp[1];
    RightLayer[2, 2] = bottomTemp[0];

    // left to bottom
    DownLayer[2, 0] = leftTemp[0];
    DownLayer[2, 1] = leftTemp[1];
    DownLayer[2, 2] = leftTemp[2];

    // top to left
    LeftLayer[0, 0] = topTemp[2];
    LeftLayer[1, 0] = topTemp[1];
    LeftLayer[2, 0] = topTemp[0];

My first comment on this is that we don't need to keep a copy of each of the layers, in fact we only have to choose one corner and one edge.
